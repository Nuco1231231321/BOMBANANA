## Button Modules in BOMBANANA!

Button modules require the Blind Monkey to press, hold, or release specific buttons — sometimes in sequence, sometimes under precise timing. The challenge: the Blind cannot see button labels or colors; they must navigate a grid by touch alone.

![Button Module Grid](/images/screenshots/ss_3ecd09d511a5ac97a7342505ec47766554d702a3.thumb.jpg)

## How Button Modules Work

A button module is a grid of buttons — typically 2×2, 3×3, or 4×3. Each button may have a color, a label, or both. The manual specifies:

- **Which button** to interact with (by position or label)
- **What action** to perform (press, hold, release, tap sequence)
- **Timing** — some buttons must be held for a specific duration
- **Sequence order** — some modules require pressing multiple buttons in order

## Button Navigation by Touch

The Blind Monkey must navigate button grids without seeing labels. The standard technique:

1. The Blind places their non-dominant hand on the top-left corner of the module
2. The Deaf describes the target button by **grid coordinates**: *"Row 2, Column 3"*
3. The Blind counts rows from top to bottom, columns from left to right by touch
4. When the Blind reaches the target button, they hold a finger on it
5. Deaf confirms visually: *"Yes, that button — press and hold for 3 seconds"*

## Button Action Types

| Action | Deaf's Instruction | Blind's Execution |
|--------|-------------------|-------------------|
| **Press** | "Press row 1, column 2" | Press and immediately release |
| **Hold** | "Hold row 2, column 1 for 3 seconds" | Press and keep holding until told to release |
| **Release** | "Release now" | Lift finger |
| **Tap sequence** | "Tap: top-left, bottom-right, top-left" | Series of quick presses |
| **Long press** | "Press and hold... keep holding... release" | Variable duration based on module |

## Common Button Module Variants

| Variant | Description | Key Challenge |
|---------|-------------|---------------|
| **Single Button** | One specific button to press | Finding it in a large grid |
| **Sequence** | Press 3-5 buttons in order | Remembering the sequence under time pressure |
| **Hold Timer** | Hold a button for exactly N seconds | The Deaf must count down; Blind cannot see timer |
| **Simon Says** | Repeat an increasingly long pattern | Memory + precision |
| **Color-Coded** | Buttons identified by color, not position | Requires color communication |

## Step-by-Step: Solving a Button Module

1. **Deaf:** *"Button module — 3×3 grid"*
2. **Mute** finds the solution in the manual
3. **Mute** gestures button coordinates (row number → column number)
4. **Deaf** confirms the target button visually
5. **Deaf:** *"Row 2, column 1 — press and hold"*
6. **Blind** navigates to the button: *"Row two... column one... holding"*
7. **Deaf** watches the module indicator: *"Keep holding... 2 seconds... release now"*
8. **Blind** releases — check for green light

## Common Mistakes

| Mistake | Prevention |
|---------|-----------|
| Blind miscounting rows/columns | Blind places anchor hand at corner, counts slowly out loud |
| Releasing hold too early | Blind only releases when Deaf says "release now" |
| Wrong button in a sequence | Deaf calls each step individually; Blind confirms each press |
| Confusing row vs column | Agree: "Row = horizontal, Column = vertical" before starting |
| Missing a timed hold window | Deaf counts down: "3... 2... 1... release" |

## Pro Tips

- **Blind:** Use the "anchor and sweep" method — keep your reference hand at the corner, sweep your other hand across rows one at a time.
- **Deaf:** When instructing a sequence, write down the steps if possible. This prevents mid-sequence confusion.
- **Mute:** For complex button sequences, break them into chunks of 2-3 buttons. Gesture chunk 1, wait for completion, gesture chunk 2.
