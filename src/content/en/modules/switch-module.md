## Switch Modules in BOMBANANA!

Switch modules feature a row of toggle switches — each switch can be flipped up or down. The manual specifies which switches must be in which position. The challenge: the Blind can feel switch positions but cannot see which way they are currently set; the Deaf must verify visual state.

![Switch Module Interface](/images/screenshots/ss_24c16a36e29f2309b796e7b2b284386c43948e69.thumb.jpg)

## How Switch Modules Work

A switch module presents 3-6 toggle switches in a row. Each switch reports its current state (up/down) visually. The manual specifies a **target configuration** — which switches should be up, which should be down.

Key mechanics:
- Each switch is independent — flipping one does not affect others
- The module may start in a random configuration
- Some switches may already be in the correct position
- Only incorrect switches need to be flipped
- Flipping a correct switch may trigger a strike

## Switch State Communication

The Blind cannot see whether a switch is currently up or down, but they CAN feel the switch's physical position. The communication pattern:

1. **Deaf** identifies which switches need flipping: *"Switches 1 and 4 need to flip"*
2. **Blind** feels the current state of each switch by touch
3. **Blind** reports: *"Switch 1 is up, Switch 2 is down, Switch 3 is up, Switch 4 is down"*
4. **Deaf** confirms visually: *"Correct. Flip switch 1 down, flip switch 4 up"*
5. **Blind** flips each switch one at a time, confirming after each

## Switch Module Variants

| Variant | Description | Strategy |
|---------|-------------|----------|
| **Set All** | All switches to one position (all up/all down) | Easiest — Deaf just says "all down" |
| **Pattern** | Specific up/down pattern (e.g., up-down-up-down) | Deaf calls each switch individually |
| **Majority Rule** | More than half must be in one position | Count current states, flip minimum switches |
| **Conditional** | Switch position depends on other modules | More complex — Mute checks conditions in manual |
| **Timed Flip** | Switches must be flipped within a time window | Speed + precision required |

## Step-by-Step

1. **Deaf:** *"Switch module — 5 switches"*
2. **Mute** locates the target configuration in the manual
3. **Mute** gestures the pattern — up/down sequence
4. **Deaf** visually identifies which switches are currently wrong
5. **Deaf:** *"Switch 2: flip down. Switch 4: flip up."*
6. **Blind** touches each switch: *"Switch 2 is currently up — flipping down... done"*
7. **Deaf** verifies: *"Switch 2 green. Now switch 4."*
8. Continue until all switches match target

## Common Mistakes

| Mistake | Prevention |
|---------|-----------|
| Flipping a correct switch | Blind names the current position before flipping |
| Blind miscounting switch positions | Number switches 1-N from left to right, always |
| Deaf giving wrong target state | Mute re-checks manual if anything looks off |
| Double-flipping (flip + flip back) | Flip once, confirm, then leave it |
| Ignoring switch labels | Some modules label switches — Deaf must read labels |

## Pro Tips

- **Blind:** Toggle switches have a distinct mechanical feel — practice identifying up vs. down by touch before the session.
- **Deaf:** Read the switches left to right and report their current state to the Blind: *"Currently: U-U-D-D-U"*. This gives the Blind an auditory map.
- **Mute:** For pattern-based switches, use a consistent up/down hand gesture — open palm raised for "up," palm facing floor for "down."
