## Symbols Modules in BOMBANANA!

Symbols modules are among the hardest puzzles in BOMBANANA! They display abstract glyphs, icons, or shapes on screen that the Blind cannot see. The Deaf must verbally describe symbols, and the Mute must match descriptions to the manual's symbol reference.

![Symbols Module](/images/screenshots/ss_71217070b739545c452accf9e600c6ca72832e64.thumb.jpg)

## How Symbols Modules Work

A symbols module displays 1-4 abstract symbols on its screen. The manual contains a lookup table mapping symbol descriptions to actions. The challenge:

- The **Blind** cannot see any symbols
- The **Deaf** must describe symbols in words that the Mute can match to the manual
- The **Mute** must identify the correct symbol from the Deaf's description, then gesture the corresponding action

## Symbol Description Strategy

The Deaf must create accurate verbal descriptions of abstract shapes. Use this framework:

1. **Basic Shape**: "It looks like a circle with a cross inside"
2. **Comparisons**: "Like a backwards letter N" or "Similar to a WiFi icon"
3. **Features**: "Three vertical lines, the middle one is taller"
4. **Common Objects**: "Looks like a crown" or "Shaped like a key"

### Standardized Symbol Vocabulary

| Visual Feature | Description Words |
|---------------|-------------------|
| Circle-based | Dot, ring, target (concentric), filled circle, empty circle |
| Line-based | Vertical line, horizontal line, diagonal, cross, X, parallel lines |
| Triangle-based | Triangle (up/down/left/right pointing), pyramid |
| Square-based | Square, rectangle, diamond, checkerboard |
| Curves | Wave, spiral, hook, crescent |
| Combined | Circle + line, square + X, triangle inside circle |

## Step-by-Step

1. **Deaf:** *"Symbols module — 2 symbols on screen"*
2. **Deaf** describes the first symbol: *"Symbol 1 looks like a circle with a dot in the center — like a target"*
3. **Mute** finds the matching symbol in the manual's lookup table
4. **Mute** gestures the corresponding action (e.g., press a specific button, cut a specific wire)
5. **Deaf** interprets the gesture and instructs Blind
6. Repeat for remaining symbols

## Common Symbol Module Variants

| Variant | Description | Key Strategy |
|---------|-------------|-------------|
| **Single Symbol** | One symbol → one action | Fastest — but symbol must be described perfectly |
| **Pair Match** | Two symbols, must find the matching pair in the manual | Deaf describes both before Mute looks up |
| **Sequence** | 3-4 symbols in order; actions form a sequence | Document each symbol-action pair as you go |
| **Exclusion** | "Which symbol is NOT present?" — harder lookup | Mute must check all possibilities |
| **Rotated/Tilted** | Symbols may appear rotated | Deaf specifies orientation: "It's turned sideways" |

## Common Mistakes

| Mistake | Prevention |
|---------|-----------|
| Vague symbol description | Be specific — "circle with a vertical line through the bottom" not just "circle with line" |
| Mismatched symbol in manual | Mute double-checks: thumbs up for "correct symbol" before gesturing action |
| Deaf guessing the action | Deaf only describes symbols — NEVER guesses what button to press |
| Missing symbol orientation | Always include rotation info: "pointing left," "upside down," "tilted 45 degrees" |
| Rushing symbol lookup | The manual lookup is the bottleneck; Mute should practice symbol recognition |

## Pro Tips

- **Deaf:** Practice describing abstract shapes before playing. Have someone show you random symbols and practice describing them in 10 words or fewer.
- **Mute:** Memorize the most common symbols in the manual. Fast recognition of the top 10-15 symbols saves crucial seconds.
- **Team:** Agree on a shared symbol vocabulary: "If I say 'target,' that means concentric circles. If I say 'plus,' that means a cross shape."
