## Wire Modules in BOMBANANA!

Wire modules are the most common — and most iconic — bomb defusal puzzle. A panel of colored wires, each one a potential solution or a potential explosion. The challenge: the Blind Monkey sees only gray; the Mute Monkey must gesture the correct color and position; the Deaf Monkey must interpret and instruct.

![Wire Module Close-up](/images/screenshots/ss_2c952b65fc5734ce79f252df9f55af35b590be4a.thumb.jpg)

## How Wire Modules Work

A wire module presents 3-8 colored wires arranged in a row or bundle. The manual specifies exactly which wire to cut based on:

- **Color** — Red, blue, yellow, green, white, black
- **Position** — First, second, third from left (or right)
- **Serial number rules** — Some wires are selected based on the bomb's serial number (even/odd last digit)
- **Neighboring wire conditions** — "Cut the blue wire only if it is next to a red wire"

## Color Communication Strategy

Since the Blind Monkey cannot see colors, your team must bridge this gap:

### Method 1: Position-Based (Simplest)
- Deaf identifies the target wire by color
- Deaf translates color → position: *"Cut the third wire from the left"*
- Blind counts by touch from left to right
- **Pros:** No special gestures needed. **Cons:** Requires wires in a clear row.

### Method 2: Color-Coded Cards
- Mute holds up a colored card matching the target wire
- Deaf sees the card color, identifies the matching wire on the bomb
- Deaf instructs the Blind by position
- **Pros:** Fast and reliable. **Cons:** Requires preparation.

### Method 3: Gesture Encoding
- Pre-agree: 1 finger = red, 2 = blue, 3 = yellow, 4 = green
- Mute holds up number → Deaf interprets color → Deaf locates wire → Blind cuts
- **Pros:** Works in all scenarios. **Cons:** Requires memorization.

## Common Wire Module Variants

| Variant | Description | Strategy |
|---------|-------------|----------|
| **Simple Cut** | One wire to cut | Fastest module — identify and cut |
| **Cut Sequence** | Cut wires in a specific order | Mute must gesture sequence; Deaf calls each step |
| **Conditional Cut** | Which wire depends on bomb serial | Mute checks serial rule in manual first |
| **Do Not Cut** | Some wires must NOT be cut | Deaf must identify forbidden wires clearly |
| **Multi-Color Striped** | Wires have multiple colors | More complex color identification needed |

## Step-by-Step: Solving a Wire Module

1. **Deaf identifies** the module type and calls: *"Wire module — 5 wires"*
2. **Mute locates** the wire module section in the manual
3. **Mute gestures** the solution — color reference first, then position
4. **Deaf interprets** gesture and visually confirms the correct wire
5. **Deaf instructs** Blind: *"Third wire from the left — cut it"*
6. **Blind confirms**: *"Third wire from left, cutting now?"*
7. **Deaf confirms**: *"Confirmed — cut"*
8. **Blind cuts** — module complete (green light) or strike (red flash)

## Common Mistakes

| Mistake | Prevention |
|---------|-----------|
| Blind cuts wrong wire (miscounting) | Blind counts out loud while touching each wire |
| Deaf misidentifies color under pressure | Use position as primary reference; color as backup |
| Mute gestures ambiguous color | Use pre-agreed color-to-number mapping |
| Cutting before confirmation | Always wait for *"Confirmed"* |
| Ignoring serial number rules | Mute checks serial-dependent conditions first |

## Pro Tips

- **Blind:** Run your finger along the wires from the anchored end (where they connect to the panel). Counting from the anchored end prevents confusion if wires cross.
- **Deaf:** If the manual requires cutting "the blue wire next to the red wire," find BOTH wires first, then confirm they are adjacent before instructing.
- **Mute:** For conditional wires, gesture the condition FIRST (e.g., point to the imaginary serial number), then the wire. This prevents the Deaf from acting on incomplete information.
