## The Mute Monkey 🙊

**Role:** *"Speak No Evil"*

The Mute Monkey holds the **defusal manual** — the single source of truth for every bomb module. You know exactly how to solve every puzzle. But you cannot speak a single word. Every solution must be communicated through **gestures, body language, and pre-agreed signals**.

![BOMBANANA! Defusal Manual Reference](/images/screenshots/ss_2c952b65fc5734ce79f252df9f55af35b590be4a.thumb.jpg)

### What You Can Do

- **Read the manual** — You have the complete, step-by-step defusal instructions for every module
- **See everything** — You can see the bomb, your teammates, and all visual information
- **Hear** — You can hear the Blind's questions and confirmations
- **Gesture** — Any body language, hand signal, or physical motion is allowed

### What You Cannot Do

- **Speak** — No words. No sounds. No voice chat. Complete vocal silence.
- **Touch the bomb** — Only the Blind Monkey can interact physically

## Your Position in the Chain

```
YOU (read manual → gesture) → DEAF MONKEY (interpret → speak) → BLIND MONKEY (execute)
```

You are the **source of truth**. Every correct solution flows from your manual through your gestures. If you gesture the wrong thing, the Deaf will say the wrong thing, and the Blind will do the wrong thing.

## The Manual: Your Most Important Tool

### Know Your Manual

Before playing, study the manual:
- Learn the **table of contents** — know where each module type is located
- Memorize **common solutions** — simple wire configurations, basic button patterns
- Understand the **decision trees** — many modules use flowcharts; learn the paths

### Manual Navigation Under Pressure

When the Deaf identifies a module:
1. Immediately find the correct section in the manual
2. Read the solution quickly but carefully
3. Plan your gestures **before** you start moving
4. Execute the gesture sequence clearly

## Building Your Gesture System

### Pre-Game Setup

Before the first bomb, spend 5 minutes with your Deaf teammate establishing:

| Information Type | Example Gestures |
|-----------------|-----------------|
| **Wire Colors** | Point to colored objects near you, or hold up color-coded cards |
| **Numbers 1-5** | Hold up fingers on one hand |
| **Numbers 6-10** | Two hands — flash 5 then add remaining fingers |
| **Directions** | Point left/right/up/down with full arm |
| **Cut** | Scissors motion with index and middle finger |
| **Press** | Index finger pushing downward |
| **Hold** | Clenched fist |
| **Release** | Open hand from fist |
| **Rotate** | Circular finger motion |
| **Flip / Toggle** | Hand flipping motion |
| **Yes / Correct** | Thumbs up |
| **No / Wrong** | Thumbs down |
| **Repeat** | Circular "again" motion |
| **Wait / Hold Position** | Open palm facing the Deaf |
| **Urgent** | Rapid, exaggerated gestures |
| **Module Complete** | Clap hands once |

### Gesture Design Principles

1. **Distinctness** — Every gesture should look clearly different from every other gesture
2. **Simplicity** — Complex gestures create errors under time pressure
3. **Consistency** — Use exactly the same gesture every time for the same meaning
4. **Visibility** — Gestures must be large enough to see even in peripheral vision

## Core Strategy: The Gesture Loop

For each module, follow this loop:

### 1. Identify
The Deaf calls out the module type. You locate the correct manual section.

### 2. Plan
Read the solution. Mentally break it into individual gestures.

### 3. Gesture, Step by Step
Execute one gesture at a time. Wait for the Deaf to nod or give thumbs up before the next gesture.

### 4. Watch the Bomb
Observe whether the Blind's actions match your intended instructions.

### 5. Correct If Needed
If the Blind does something wrong, use thumbs down immediately and restart the gesture sequence.

## Advanced Gesture Techniques

### Gesture Chaining
Combine gestures into fluid sequences for speed:
- Color gesture → Number gesture → Position gesture → Action gesture
- Example: Point to blue → Hold up 3 fingers → Point left → Scissors motion
- Translation: "Cut the third blue wire from the left"

### Spatial Referencing
Use your own body to represent the bomb layout:
- Point to your left arm = left side of bomb
- Touch your chest = center of bomb
- Raise hand high = top of bomb

### Temporal Signaling
Communicate timing through gesture speed:
- Fast, repeated gesture = "Do this now, urgently"
- Slow, deliberate gesture = "Careful, precise action needed"
- Gesture held static = "Hold this position"
- Rapid release of static gesture = "Release now"

## Module Gesture Patterns

### Wire Cutting Modules
```
1. Point to color reference → 2. Hold up number of wire position → 3. Scissors gesture
```

### Button Press Modules
```
1. Point to color reference → 2. Show row number → 3. Show column number → 4. Press gesture
```

### Sequence Modules
```
1. Hold up "Step 1" finger → 2. Gesture action for step 1
3. Wait for completion signal
4. Hold up "Step 2" fingers → 5. Gesture action for step 2
...repeat...
```

### Timer-Based Modules
Add urgency signaling:
- Gesture at normal speed = "Plenty of time"
- Accelerate gestures = "Timer is running low"

## Common Mistakes & Recovery

| Mistake | What Happens | Recovery |
|---------|-------------|----------|
| Gesturing too small | Deaf cannot see | Exaggerate — arms fully extended |
| Gesturing too fast | Deaf misses key information | Slow down — one gesture per 2 seconds |
| Wrong manual section | Incorrect solution | Thumbs down vigorously, find correct section |
| Ambiguous gesture | Deaf guesses wrong meaning | Use the "repeat" signal, find a clearer gesture |
| Panic freezing | Information stops flowing | Take a breath, find one thing you can gesture, start there |

## Team Optimization

### Pre-Session Checklist
- [ ] Review manual table of contents
- [ ] Place manual in easy reach
- [ ] Agree on gesture vocabulary with Deaf
- [ ] Practice top 5 most common gestures
- [ ] Confirm feedback signals (yes/no/repeat/urgent)
- [ ] Position yourself where the Deaf has a clear, unobstructed view

### Communication Efficiency Tips
- **Lead the Deaf's attention** — Point at the module you are referencing before gesturing the solution
- **Use pre-agreed shortcuts** — For common module types, agree on a single "module type" gesture
- **Confirm understanding** — After each gesture sequence, wait for the Deaf's thumbs up before proceeding
- **Stay visible** — Don't let your hands drop below chest level where the Deaf might not see them
